# Clinical Document Intelligence — Live Demo

**EMNLP 2026 System Demonstration**

A clinical document intelligence platform that extracts structured entities
from unstructured NHS clinical letters (referrals, clinic letters, discharge
summaries), detects safety-relevant flags (allergy conflicts, overdue
follow-ups, duplicate medications), and identifies contradictions across a
patient's document history.

This repository contains a **self-contained demo** with two pre-computed
synthetic patients. No API keys, no cloud credentials, no external services
required.

## Quick start

Requires [Docker](https://www.docker.com/products/docker-desktop/) and
Docker Compose (bundled with Docker Desktop).

```bash
git clone <this-repo-url>
cd <this-repo>
docker-compose up
```

Wait ~30 seconds for both containers to start, then open:

**http://localhost:3001**

## What you'll see

Two synthetic patients, fully pre-processed through the pipeline:

- **Margaret Thompson** (patient_001) — cardiology/CKD case with a planted
  allergy contradiction (NKDA recorded in two documents, penicillin allergy
  recorded in a third). Demonstrates the contradiction-detection agent and
  the flag-identity matcher.

- **Daniel Ofori** (patient_002) — type 2 diabetes case with **zero**
  planted contradictions, used as a false-positive control to confirm the
  system doesn't hallucinate disagreements on a consistent chart.

Explore:
- **Briefing** — clinical summary card (conditions, medications, open flags)
- **Flags** — severity-tagged safety flags with source document linkage
- **Contradictions** — side-by-side conflicting claims with explanation
- **Timeline** — chronological event view, filterable by type
- **Documents** — the underlying synthetic clinical letters

## Architecture

```
Browser (Next.js) → FastAPI demo backend → pre-computed JSON fixtures
```

The demo backend (`demo/api/main.py`) serves pre-computed pipeline outputs
from `demo/fixtures/*.json`. These fixtures were generated by running the
**full production pipeline** (scispaCy NER → rule-based flag detection →
LLM-assisted contradiction detection) against synthetic clinical documents,
then exporting the results. This means what you see is real system output,
not mocked data — just pre-computed so the demo requires no external API
keys or cloud services to run.

## Performance

On the underlying pipeline (measured on the source documents, not this demo
layer): ~0.2–1.3 seconds per document for full NER + flag extraction on
commodity hardware (see paper for details).

## Repository structure

```

  Dockerfile.api         — API container
frontend/               — Next.js UI (shared with production system)
docker-compose.yml       — one-command orchestration
```
